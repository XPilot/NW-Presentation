A little about NodeWebkit...

  • It's an app runtime based on Chromium
  • It's powered by Webkit browser engine (DUH!)
  • It's created and developed by Intel Open Source Technology Center

Not another HTML2Exe...

  • It's not a HTML2Exe converter
  • It's not what Adobe AIR is to Flash
  • It's not a tool for Power Point presentations (except this one)

Not another HTML2Exe...

  • Think of it as a pocket browser on steroids
  • That is flexible enough to do low level stuff  
  • And it's cross compatible* on Windows Linux and OSX
  • Best of all, you can use any node module

*Some Node packages may not be compatible and will most likely crash your app!

Why should I use NW?

Let's get started...

 

//
// Package.json
// REMOVE COMMENTS BEFORE RUNNING!
//
{
  "name": "helloWorld",
  "version": "0.0.1",
  "description": "A Hello Node webkit app ",
  // define entry point
  "main": "index.html",
  "window": {
    // URL Toolbar (used for debugging)
    "toolbar": true,
    "width": 1024,
    "height": 768
  },
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "start": "node_modules/.bin/nw"
  },
  "author": "",
  "license": "MIT",
  "dependencies": {
    "nw": "^0.12.2"
  }
}

How does it work?

 

1) NW reads the package.json file, looking for specific tags (note that dependencies are not installed automatically)

2) Finds the main entry point (index.html) and loads it.

3) That's it!

Why should I use NW?

 

Well it's like asking why should I use React over Angular and vice versa.

  • entry point: web page  vs  javascript
  • build system: Full Chromium implementation vs libchromiumcontent
  • node integration: hacked vs looped messages
  • simplicity vs complexity

Thing is... it's not about the tool, it's about the result...
